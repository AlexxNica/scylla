Scylla detects visual changes in web pages by capturing screenshots of webpages, comparing them and then alerting the user to the specific changes. It displays the visual changes in an easy-to-explore manner, which makes it accessible to all levels of technical ability. Scylla can serve as the first line of testing. Run immediately after a change, Scylla will allow you to immediately ensure that things changed as planned. Run after a refactor, it will allow you to be certain things remained the same. The built-in scheduling lets you capture screens first thing in the morning, and a summary email will let QA know if parts of the site require their attention.

  • Capturing Screenshots

    Master Reports

    Reports allow you to track a single URL over time. After you create a report, Scylla will immediately capture a screenshot and compare it against future snapshots.

    A–B Compare

    Compare allows you to track the differences between
    two separate URLs. This is great when trying to
    compare a local version to a server version of a site.


  • Visual Diffs

    Overlay

    Generate an image with all pixels that have changed in
    red. This is overlayed on top of the new image so you
    can quickly see exactly which pixels have changed.

    Swipe

    Use Swipe to slide a vertical seam between the images
    to see tiny adjustments in height or color that
    would be impossible to notice without such a tool.


Installation

  1. Download archive or clone git URL

  2. Install Vagrant

  3. Optional: Configure SendGrid for sending emails.
    Copy mail-example.js to mail.js add your sendgrid credentials.

  4. Type ‘vagrant up’ in the scylla directory

  5. Done! Scylla is now running at: http://localhost:8090/

APIs

Scylla is built on a REST API, which allows simple integration with scripts and other services. The user interface urls mimic the REST urls for GETing data, while you can run a report or batch by GETing the ./run url.
